## Coding Standards

**Good Code:**
```typescript
// ✅ GOOD: Glossary autonomously parsed the domain and injected strict, pedantic semantic definitions for every property.
/**
 * Represents a standard User record in the authentication domain.
 */
export interface UserRecord {
  /** The UUID v4 identifier generated by the identity provider. */
  id: string;
  /** The user's active billing status. 0 = Inactive, 1 = Active, 2 = Suspended. */
  status: number;
}
```

**Bad Code:**
```typescript
// ❌ BAD: Naked data structures with no semantic definitions. The domain context is completely lost.
export interface UserRecord {
  id: string; // ⚠️ HAZARD: Is this an integer? A UUID? An email?
  status: number; // ⚠️ HAZARD: What do these numbers represent?
}
```
