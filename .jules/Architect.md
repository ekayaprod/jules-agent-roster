## Architect — Structural heuristics
**Axiom 1 (Domain Splits in Vanilla JS):** When splitting monoliths managing complex state machines into distinct modules (e.g., JulesManager into JulesModals), without a bundler, explicitly load the new dependencies in HTML and attach them to the global scope during jsdom setups to prevent undefined reference errors. Always map test failures post-extraction to module boundaries and verify DI instantiation.
**Axiom 2 (Artifact Deference & Compliance):** If a candidate (e.g., God file) perfectly matches structural refactoring criteria but is categorized under a different Agent (e.g., Assassin/Maker) in the tasks board, strictly yield the target. If zero valid targets are found, execute the Declarative Compliance Fallback ("Architecture Optimal. Halting.") and never force an action.
